Syslog级别详解

2024-01-23 1 分钟阅读

首先是：

[facility].[priority]

facility表示设备、设施，特别装置：

 1auth      # 认证相关的  
 2authpriv  # 权限,授权相关的  
 3cron      # 任务计划相关的  
 4daemon    # 守护进程相关的  
 5kern      # 内核相关的  
 6lpr       # 打印相关的  
 7mail      # 邮件相关的  
 8mark      # 标记相关的  
 9news      # 新闻相关的  
10security  # 安全相关的,与auth 类似  
11syslog    # syslog自己的  
12user      # 用户相关的  
13uucp      # unix to unix cp 相关的  
14local0 到 local7 # 用户自定义使用  
15*         # *表示所有的facility

一定要注意上面的，local0 到 local7 # 用户自定义使用

priority表示优先权，日志优先权(log level),一般有以下几种优先权(从低到高)：

 1debug           # 程序或系统的调试信息  
 2info            # 一般信息,  
 3notice          # 不影响正常功能,需要注意的消息  
 4warning/warn    # 可能影响系统功能,需要提醒用户的重要事件  
 5err/error       # 错误信息  
 6crit            # 比较严重的  
 7alert           # 必须马上处理的  
 8emerg/oanic     # 会导致系统不可用的  
 9*               # 表示所有的日志级别
10none            # 跟* 相反,表示啥也没有

syslog的日志优先权默认是info，这时候用syslog为debug(最低日志优先权)来写日志，syslog服务是不会写入日志的。
